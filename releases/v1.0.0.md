# Édition Canonique Stabilisée
## Référentiel Documentaire IA 
### Version 1.0.0

---

# 12.1 Objet

Le présent document constitue l'acte de stabilisation officiel du Référentiel Documentaire IA 40.x.

Il définit :

* le périmètre officiel du référentiel ;
* les documents constitutifs ;
* les règles de maintenance ;
* les critères de compatibilité ;
* les règles de versionnement ;
* la version de référence.

---

# 12.2 Portée officielle

Le Référentiel Documentaire IA 40.x s'applique à :

* la validation documentaire ;
* la validation de citations ;
* la validation de sources ;
* la validation d'études ;
* l'analyse documentaire ;
* la production de corpus ;
* la production de synthèses documentaires ;
* les systèmes de gestion documentaire ;
* les assistants IA ;
* les systèmes RAG ;
* les bases de connaissances.

---

# 12.5 Principes fondateurs

Le référentiel repose sur les principes suivants.

## Vérifiabilité

Toute information importante doit pouvoir être vérifiée.

---

## Exactitude

Les conclusions doivent demeurer compatibles avec les sources.

---

## Traçabilité

L'origine documentaire doit pouvoir être identifiée.

---

## Transparence

Les limites et incertitudes doivent être documentées.

---

## Complétude documentaire

Le niveau d'achèvement documentaire doit être indiqué.

---

## Réutilisabilité

Les résultats doivent pouvoir être réemployés dans d'autres contextes.

---

## Uniformité

Les règles documentaires doivent être appliquées de façon cohérente.

# 12.8 Compatibilité documentaire

Une implémentation est compatible avec le référentiel lorsqu'elle :

* applique les sections pertinentes ;
* respecte les formats officiels ;
* respecte les définitions officielles ;
* respecte les critères de conformité.

---

# 12.9 Compatibilité descendante

Une version future doit conserver la compatibilité avec :

```text
Version 1.0.0
```

sauf justification explicite.

---

# 12.10 Gestion des modifications

Les modifications sont classées selon trois catégories.

---

## Corrective

Correction sans impact fonctionnel.

Exemples :

* faute ;
* erreur de renvoi ;
* erreur de numérotation.

---

## Évolutive

Ajout compatible.

Exemples :

* nouveau modèle ;
* nouvelle matrice ;
* nouveau champ documentaire.

---

## Majeure

Modification pouvant affecter la compatibilité.

Exemples :

* suppression d'un format ;
* modification d'une définition ;
* suppression d'un niveau documentaire.

---

# 12.11 Gestion des versions

Format officiel :

```text
MAJEUR.MINEUR.CORRECTIF
```

Exemple :

```text
1.0.0
1.1.0
1.1.1
2.0.0
```

---

# 12.12 Critère de stabilité

Une édition est considérée stable lorsque :

* les dépendances sont résolues ;
* les formats sont définis ;
* les modèles sont documentés ;
* les matrices sont présentes ;
* les mécanismes de conformité sont définis.

---

# 12.13 Critère d'autonomie

Le référentiel est autonome lorsqu'une IA peut :

* interpréter les demandes ;
* sélectionner les directives ;
* produire les formats requis ;
* évaluer la conformité ;

sans dépendance à une documentation externe.

---
